Decisions adopted while drafting plans/start.md (Stage 1):

API client: `@hey-api/openapi-ts` with the `@tanstack/react-query` plugin generates a
typed client + hooks into `frontend/src/api/generated/` (gitignored). Driven solely by
`openapi-ts.config.ts` — the `gen-api` script invokes `openapi-ts` with no CLI overrides.
Backend `operationId`s become hook names (`useUsersListQuery`, `useBookCreateMutation`,
…). Run `just gen-api` against a running backend; CI also runs it and fails on diffs.

Auth client: BetterAuth via `better-auth/react`'s `createAuthClient`. All API calls send
`credentials: 'include'`. A single global response interceptor handles 401:
- One in-flight `getSession()` refresh is shared by N concurrent 401s.
- If session is restored, the original request retries exactly once.
- Otherwise: clear TanStack Query cache, toast "Sesja wygasła", redirect to
  `/login?next=<currentPath>`. No redirect loop on `/login`.
While the editor is mounted, `useSessionPing()` polls `getSession()` every 4 minutes
when the tab is visible to keep the rolling session alive.

Layout principle: the editor center is a fixed A4 page (~794 px wide). It does NOT
shrink on narrow viewports. Side panes collapse to icon rails (`lg`/`xl`), then to
overlay drawers (`md`), then hide entirely (`sm`). On mobile the page keeps A4 width and
the viewport scrolls horizontally. Pane open/close state persists per-user in
localStorage via Zustand.

i18n: `react-i18next` with three locales — `pl` (Polish, primary/default), `en`
(English, developer fallback), `ua` (Ukrainian; project convention uses `ua` rather
than ISO `uk` to avoid "UK English" confusion). Resources under
`src/locales/{pl,en,ua}/<namespace>.json`. Namespaces: common, auth, admin, coordinator,
editor, errors. `pl` is source-of-truth; CI script `scripts/check-locales.ts` fails on
key drift. A typed `useT(namespace)` wrapper plus a generated `resources.d.ts`
(emitted by `scripts/gen-locale-types.ts`) make missing keys a compile-time error.
Language switcher persists choice to localStorage and `user.preferredLocale` via
`PATCH /api/me`. Backend error codes are translated via `errors.json`.

Editor port: `editor-poc/src/*` is copied wholesale into `frontend/src/editor/`.
`identity/storage.ts` localStorage-based identity is replaced with the BetterAuth
session + per-book role from the assignments query. `collab/yDoc.ts` swaps
`WebrtcProvider + IndexeddbPersistence` for `HocuspocusProvider` (IndexedDB cache kept
for offline flush). Tiptap schema-bearing extensions come from the workspace package
`@przeswity/editor-schema`; only browser/interactive extensions are added on top.

People/roles UX: a `<PeoplePicker>` dialog (typeahead with avatars, competency tags,
chips with per-chip role select, "Add another role" to assign the same person multiple
roles) plus a `<AssignmentList>` component (inline change-role dropdown, remove with
confirm, "Ty" pin). Used in book create form, book settings, and the editor's right-pane
"Osoby" tab. Bulk-submit with optimistic updates and rollback on failure.

Settings: `/settings` (Profile incl. language switcher), `/settings/password`,
`/settings/sessions` (BetterAuth listSessions/revokeSession).

Quality bar: skeleton loaders, friendly empty states, toast feedback on every mutation,
shadcn AlertDialog for destructive actions, error boundary, 404 page, full keyboard a11y,
shortcut modal already from editor-poc, print stylesheet that strips chrome.
